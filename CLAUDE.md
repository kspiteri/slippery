## Architecture

**Slippery** is a React 19 + TypeScript PWA that helps Bergen cyclists assess whether it's safe to ride their commute based on route conditions and weather.

### Data flow

1. `AddressForm` collects from/to/waypoints with autocomplete (ORS Geocoding + Nominatim fallback)
2. `App.tsx` orchestrates: geocode → fetch route (ORS Directions) → fetch weather (MET Norway) → score
3. `slipperiness.ts` computes a 0–100+ risk score from temperature, precipitation, surface type, and active MetAlerts
4. `Verdict` renders results across three time horizons: now / +2h / +8h

### Key directories

- `src/api/` — ORS (routing + geocoding), MET Norway (forecast + alerts), and elevation grid builder
- `src/logic/slipperiness.ts` — risk scoring algorithm; score thresholds: 0–25 clear, 26–55 caution, 56–79 high, 80+ don't ride; studded tyres subtract 35 before re-evaluating
- `src/ui/` — `AddressForm`, `Verdict`, and ASCII elevation canvas renderer
- `src/i18n/` — EN/NO translations via i18next; language stored in `localStorage` (`slippery_lang`)
- `src/state.ts` — saved from/to/waypoints in `localStorage` (`slippery_addresses`)

### Deployment

GitHub Actions deploys `main` to GitHub Pages. Vite is configured with `base: '/slippery/'` for the subpath. The PWA manifest and service worker are generated by `vite-plugin-pwa`.
